# 变更日志

## [2026-03-06] 619b01d

### 新增功能
- **事件系统** - 引入 `EventBus` 事件总线，支持发布/订阅模式、通配符监听、事件历史记录
- **插件架构** - 新增 `BasePlugin` 抽象基类和 `PluginRegistry` 插件注册表，支持功能扩展
  - `NotificationPlugin` - 任务事件通知插件，支持日志文件记录
  - `ExportPlugin` - 任务导出插件，支持 JSON/CSV/Markdown 格式
- **任务模板系统** - 新增 `TaskTemplate` 和 `TemplateRegistry`，支持变量插值
  - 内置模板：`bug-fix`（Bug修复）、`feature`（功能开发）、`release`（版本发布）
- **批量操作** - 新增 `BatchOperations` 类，支持：
  - 批量任务导入/导出（JSON格式）
  - 批量状态变更（完成/取消/重分配）
  - 重复任务检测（基于Jaccard相似度）
- **标签系统** - Task 模型新增 `tags` 字段，支持任务标签分类
- **CLI功能扩展** - 新增多个命令组：
  - `template list/use` - 模板管理和使用
  - `batch complete/cancel` - 批量操作
  - `import/export` - 数据导入导出
  - `duplicates` - 重复任务检测
  - `history` - 事件历史查看
  - `plugins` - 插件状态查看

### 功能增强
- **事件驱动** - 所有任务操作现在都触发相应事件（创建、完成、取消、删除、分配）
- **CLI增强** - `add` 命令支持 `--tags` 参数，`list` 命令显示标签和进度信息
- **系统集成** - 事件系统与插件系统完全集成，支持实时响应

### 架构改进
- **版本升级** - 项目版本从 0.2.0 升级到 0.3.0
- **模块化设计** - 新增 `events.py`、`templates.py`、`batch.py` 和 `plugins/` 目录
- **向后兼容** - 所有原有API保持兼容，仅在 Task 模型中增加 `tags` 字段
- **文档完善** - 项目描述更新为"支持持久化存储、事件系统、插件架构和批量操作"

### 内置资源
- **模板库** - 提供3个企业级内置模板，涵盖常见工作流程
- **插件示例** - 2个功能完整的内置插件作为开发参考
- **事件类型** - 定义11种系统事件类型，覆盖完整的任务生命周期

## [2026-03-06] b21f856

### 新增功能
- **存储抽象化** - 引入 `BaseTaskStore` 抽象接口，解耦存储实现
- **JSON持久化存储** - 新增 `JsonStore` 类，支持数据持久化到JSON文件
- **子任务支持** - 添加 `SubTask` 模型，支持任务分解和进度跟踪
- **任务状态扩展** - 新增 `CANCELLED` 状态
- **优先级扩展** - 新增 `CRITICAL` 优先级别
- **截止日期** - 任务支持 `due_date` 字段和逾期检测
- **任务搜索** - CLI 新增 `search` 命令，支持关键词搜索
- **统计功能** - 新增 `stats` 模块和CLI命令，提供任务统计报表
- **任务删除** - CLI 新增 `remove` 命令，支持任务删除
- **序列化支持** - Task 类新增 `to_dict()` 和 `from_dict()` 方法

### 功能改进
- **CLI参数扩展** - `add` 和 `list` 命令支持 `--assignee` 参数
- **存储接口统一** - 所有存储实现继承统一的抽象基类
- **任务属性丰富** - 支持子任务、截止日期、进度计算

### 架构调整
- **模块重命名** - `TaskStore` 重命名为 `MemoryStore`
- **依赖注入** - CLI 通过参数传递存储实例，提高可测试性
- **职责分离** - 统计功能独立为单独模块

### 技术改进
- **类型支持** - 增强类型注解和类型检查
- **错误处理** - 改进CLI错误处理和用户提示
- **代码质量** - 重构命令处理器，统一函数签名
